# Scene Understanding Platform — 项目设计文档

> 本文档描述平台的目标架构与**当前实现状态**。快速上手与 API 示例见 [README.md](./README.md)。

---

## 1. 项目概述

**Scene Understanding Platform** 是一个面向视觉定位（Visual Localization）与场景理解的 Web 平台。当前版本已具备场景/数据集管理、模型管理、**单图推理 Hub** 与 **场景推理 Hub**，并接入 SALAD、NetVLAD、SuperPoint、SIFT、LightGlue、SAM 等真实推理能力。

```

### 当前已实现链路

**单图推理：**

```
Query Image → [图像检索 | 双图匹配 | 关键点可视化 | SAM 交互分割]
```

**场景推理：**

```
数据集 split (rgb/) → [批量特征提取 | SAM 批量语义分割] → 输出目录 + 进度轮询
```

**目标用户：** 研究人员与工程师管理多场景数据集、调用多种 CV 模型完成检索/匹配/特征/分割等实验，并为后续端到端定位评测打基础。

## 3. 系统架构

### 3.1 当前架构（已实现）

```
┌──────────────────────────────────────────────────────────────────┐
│                    Frontend (React + Vite)                        │
│  场景管理 │ 模型管理 │ 单图推理 Hub │ 场景推理 Hub                  │
└────────────────────────────┬─────────────────────────────────────┘
                             │ HTTP
┌────────────────────────────▼─────────────────────────────────────┐
│                     FastAPI (backend/app)                         │
│  /scenes  /models  /inference  /benchmarks  /media               │
└──────┬─────────────────────────────┬───────────────────────────────┘
       │                             │
┌──────▼──────┐              ┌───────▼────────────────────────────┐
│ SQLite      │              │ Plugin Registry + Engines           │
│ scenes 表   │              │ Retriever / Extractor / Matcher /     │
│             │              │ Segmenter / SemanticSegmenter       │
└─────────────┘              └────────────────────────────────────┘
       │
┌──────▼──────────────────────────────────────────────────────────┐
│ 本地文件系统                                                      │
│ datasets/  models/  data/features/  data/segmentation/  uploads/ │
└──────────────────────────────────────────────────────────────────┘
```

**设计原则：**

- API 层负责路由、校验、任务创建与进度查询；推理逻辑在 `services/` 插件中
- 场景推理任务在**后台线程**执行，内存中维护 `job_id → 进度`（重启后丢失）
- 大文件（图像、权重、特征索引）存本地路径，DB/JSON 仅存元数据与路径


## 4. 功能模块

### 4.1 场景管理 ✅

**页面：** `/scenes`

**能力：**

| 能力 | 状态 | 说明 |
|------|------|------|
| 内置数据集展示 | ✅ | CrossLoc（4 场景）、UAVD4L（2 场景） |
| 自定义数据集注册 | ✅ | 扫描目录、校验结构、写入 `custom_datasets.json` |
| 自定义数据集删除 | ✅ | 同步删除 SQLite 中对应场景行 |
| 场景网格与 SQLite 列表 | ✅ | 注册状态、参考图路径、特征索引路径 |

**双轨 Catalog：**

- **内置**：`services/dataset_catalog.py` 硬编码
- **自定义**：`services/dataset_registry.py` 持久化
- **合并**：`get_all_dataset_families()` → `GET /scenes/catalog` → 推理页级联下拉

**自定义目录约定：**

```
DatasetRoot/
├── SceneA/
│   ├── train/  { calibration/, poses/, rgb/ }
│   └── test/   { calibration/, poses/, rgb/ }
└── SceneB/ ...
```

- `family_id` = 根目录 basename（slug 化）
- `dataset_id` = `{family_id}-{scene_id}-{train|test}`

**Scene 表核心字段（SQLite）：**

```
Scene {
  id, name, description
  dataset_family, scene_slug
  reference_images_dir      // 默认 train/rgb
  feature_index_path        // 默认 data/features/{dataset_id}
  train_image_count, test_image_count
  point_cloud_path          // 预留
  status, created_at, updated_at
}
```

**相关 API：**

- `GET /scenes/catalog`
- `POST /scenes/datasets/scan`、`POST /scenes/datasets/register`
- `DELETE /scenes/datasets/{family_id}`
- `GET /scenes/datasets/browse`
- `GET/POST/PATCH/DELETE /scenes/{id}`

---

### 4.2 模型管理 ✅

**页面：** `/models`

**内置模型（6 个）：**

| model_id | 名称 | 能力 |
|----------|------|------|
| `salad` | SALAD | 特征提取、图像检索 |
| `netvlad` | NetVLAD | 特征提取、图像检索 |
| `superpoint` | SuperPoint | 特征提取、关键点检测 |
| `sift` | SIFT | 特征提取、关键点检测 |
| `lightglue` | LightGlue | 图像匹配 |
| `sam` | SAM (ViT-B) | 交互分割、语义分割 |

**生命周期：**

```
内置注册 → 推理时自动 load → 卸载(DELETE) → removed_models.json 记录 → 重启/sync 可恢复
```

自定义模型注册 API 存在，但**可运行插件动态接入**仍在规划中。

**相关 API：**

- `GET/POST /models`
- `POST /models/{id}/load`、`POST /models/{id}/unload`
- `DELETE /models/{id}`

---

### 4.3 单图推理 Hub ✅

**入口：** `/inference`

子页面左上角 **「功能选择」** 下拉可在 Hub 内切换，无需返回上级。

| 模块 | 路径 | 说明 | 状态 |
|------|------|------|------|
| 图像检索 | `/inference/retrieval` | Top-8 相似图检索（SALAD / NetVLAD） | ✅ |
| 图像匹配 | `/inference/matching` | 双图匹配可视化（LightGlue） | ✅ |
| 可视特征 | `/inference/feature-visualization` | 局部关键点可视化（SuperPoint / SIFT） | ✅ |
| 交互分割 | `/inference/interactive-segmentation` | 点提示 SAM 分割 | ✅ |

**图像检索流程：**

1. 选择数据集 / 场景 / split（`useDatasetSceneSelection` → `/scenes/catalog`）
2. 上传 Query 图
3. `POST /inference/run`，`pipeline_stages=["image_retrieval"]`
4. Retriever 读取预计算特征或 on-the-fly 建库，返回 Top-K 参考图

**交互分割流程：**

1. 上传图像 → `POST /inference/interactive-segmentation/session`
2. 点击前景点 → `POST /inference/interactive-segmentation/predict`
3. 返回 mask 轮廓与 score

---

### 4.4 场景推理 Hub ✅

**入口：** `/scene-inference`

| 模块 | 路径 | 说明 | 状态 |
|------|------|------|------|
| 特征提取 | `/scene-inference/feature-extraction` | 批量提取参考图特征 | ✅ |
| 语义分割 | `/scene-inference/semantic-segmentation` | SAM 自动掩码，保存 mask + manifest | ✅ |

**任务模式：**

1. 选择数据集 split → 指定 `dataset_id` 与 `output_path`
2. `POST /benchmarks/feature-extraction/run` 或 `.../semantic-segmentation/run`
3. 轮询 `GET /benchmarks/.../ {job_id}` 获取 `progress`（0–100）

**特征提取输出：** `data/features/{dataset_id}/`（描述子索引）

**语义分割输出：** `data/segmentation/{dataset_id}/`（`{stem}_mask.png` + `manifest.json`）

## 5. 插件化推理架构 ✅

```
model_id → plugins/registry.py → 具体 Plugin 实现
```

| 插件类型 | 注册表 | 目录 | 用途 |
|----------|--------|------|------|
| `Retriever` | `RETRIEVER_REGISTRY` | `retrievers/` | 单图 Top-K 检索 |
| `FeatureExtractor` | `FEATURE_EXTRACTOR_REGISTRY` | `feature_extractors/` | 场景批量特征 |
| `KeypointDetector` | `DETECTOR_REGISTRY` | `detectors/` | 关键点检测 |
| `Matcher` | `MATCHER_REGISTRY` | `matchers/` | 双图匹配 |
| `Segmenter` | `SEGMENTER_REGISTRY` | `segmenters/` | 点提示交互分割 |
| `SemanticSegmenter` | `SEMANTIC_SEGMENTER_REGISTRY` | `semantic_segmenters/` | 批量语义分割 |

**Engine 单例：** 各模型对应 `*Engine.get_shared()`，避免重复加载权重。

**扩展新内置模型步骤：**

1. 实现 Engine + Plugin 类
2. 在 `registry.py` 注册
3. 在 `model_service._build_default_models()` 增加元数据与 `capabilities`
4. 前端按需增加能力筛选（`modelsForStage`）

## 6. 数据集与 Catalog 设计 ✅

### 6.1 数据流

```
dataset_catalog.py (内置)
        ↘
         get_all_dataset_families() → build_dataset_catalog() → 前端下拉
         mock_datasets._DATASET_BY_ID  → 推理解析 rgb_dir / dataset_id
        ↗
dataset_registry.py (自定义 JSON)
        ↘
scene_seed.seed_catalog_scenes() → SQLite Scene 行
```

### 6.2 内置数据集

| 数据集 | 场景 | 目录约定 |
|--------|------|----------|
| CrossLoc | nature, natureoop, urban, urbanoop | `{scene}/{train\|test}/rgb/` |
| UAVD4L | intraj, outtraj | `{traj}/{train\|test}/rgb/` |

路径在 `dataset_catalog.py` 配置，部署时需改为本机路径。

### 6.3 推理页数据集选择

- 组件：`DatasetSceneFields` + `useDatasetSceneSelection`
- 用于：图像检索、场景特征提取、场景语义分割
- 传参：`scene_id`（scene_slug）+ `dataset_id`（如 `crossloc-nature-test`）

## 13. 非功能需求

| 项 | 当前 | 目标 |
|----|------|------|
